Copilot spend is governed by budgets at four levels that work together, not as alternatives. This page shows the order they're evaluated in and which one wins when they overlap.
How a single request is evaluated
Copilot AI request that consumes credits
1
User-level budget (ULB) — checked first, always a hard stop
Over the user's ULB → ✕ Blocked — even if the cost center still has budget.
Within ULB → continue ↓
2
Is there credit left in the shared pool?
Pool phase → ✓ Drawn from the pool, no metered charge.
Pool exhausted → metered phase ↓
3
Metered caps, in order: cost center → organization → enterprise
Over any active cap with hard-stop on → ✕ Blocked.
Cap is alert-only → billed anyway, notification sent.
Which ULB value applies
Individual overridewins
Set for one specific user.
▲ overrides
Cost-center user-level budget
One per-user amount for every member of a cost center.
▲ overrides
Universal ULB
Default that applies to every licensed user.
Most specific wins. An individual override replaces the cost-center and universal ULB for that one person.
Two phases per cycle
Pool phase
Everyone draws from the shared included-credit pool. Only the ULB is active — cost-center / org / enterprise caps do nothing yet.
Metered phase
Pool exhausted; further usage is billed. ULB still applies, and now the cost-center → org → enterprise metered caps apply too.
Rules that surprise people
ULB is checked first
A user over their ULB is blocked even if their cost center has plenty of budget left.
A cost-center budget doesn't extend a ULB
It caps the group's metered spend; it never raises an individual's per-user limit.
Starved enterprise budget
Raising ULBs without raising the enterprise cap can make the enterprise budget block users before they reach their own limit.
Alert-only ≠ hard stop
Enterprise / cost-center caps only block when "stop usage" is enabled. ULBs always hard-stop; a $0 ULB blocks immediately.
